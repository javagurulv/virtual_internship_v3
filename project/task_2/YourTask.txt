В этом задании вы научитесь собирать (билдить)
проект с помощью команд из закладки Gradle.

В данном проекте для сборки проекта используется Build Tool Gradle.
Его закладку вы найдёте справа в IntelliJ IDEA. Откройте ее.

Важная функциональность в Gradle закладке это символ с двумя стрелочками
Reload all Gradle projects - обязательно используйте его после добавления
в проект новых зависимостей.

Запустить полную сборку проекта можно выполнив команду
Tasks -> build -> build в окне Gradle. Эта команда запустит
сборку всех модулей в проекте. Выполнение этой команды может
занять определённое время, так как модулей в проекте достаточно много.

Запустите полную сборку проекта!

Для того чтобы выполнить сборку только вашего модуля в окне Gradle
выберите название internship модуля и запустите команду:
internship -> Tasks -> build -> build.

Запустите сборку вашего модуля!

Запуск команд Gradle из терминала обеспечивается
следующими файлами (скриптами) в нашем проекте:
 - gradlew  - для unix систем
 - gradlew.bat - для windows
Для правильной работы этим скриптам требуется
наличие установленного параметра окружения JAVA_HOME.
То есть эти скрипты используя JAVA_HOME находят
путь до установленной на вашем компьютере JDK,
в ней находят JVM и с помощью нее запускают
Gradle wrapper (как Java библиотеку). Без
прописанной JAVA_HOME выполнить команды Gradle
из терминала не получится.

Ваше задание: узнайте какие действия выполняет команда build в Gradle.

файл /documentation/Task_x_description.txt. Формат заполнения этого
файла описан в task_1 в файле Learning_reflection.txt.
